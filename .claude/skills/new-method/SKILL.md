# 새 논문 메소드 추가 절차

MemoryOS(첫 메소드)에서 수렴한 순서. 한 번에 진행하지 말고 단계마다
사용자와 합의한다.

1. **논문이 명세서.** PDF에서 명세를 직접 추출해 구조·수식·상수·프롬프트
   유무를 정리하고 사용자와 확인한다. 논문에 없는 것은 "없음"으로 명시 —
   코드에만 있는 메커니즘을 논문 것으로 착각하지 않는다.
2. **reference 고정.** 원본 repo를 `scripts/fetch_reference.py`에 SHA 핀으로
   추가한다 (`external/<이름>`, 커밋 금지). 원본과 논문의 괴리를 실측으로
   확인해 기록한다 — MemoryOS는 구현체 3개 전부 논문과 달랐다.
3. **소켓에 맞춰 구현.** `methods/<이름>/` 안에만. 고유 스키마를 공용 계층에
   유출하지 않는다. 해석·결정은 해당 파일 docstring에. 논문이 침묵하는
   상수·프롬프트만 원본에서 차용한다. 소켓(`MemoryMethod`)이 안 맞으면
   설계 신호 — 먼저 합의.
4. **dry-run으로 시나리오 검증.** 실제 LoCoMo 데이터 + grounded fake로 예상
   동작을 실측한 뒤에 본 실험·가이드를 설계한다. 머리로 짠 시나리오는
   틀린다 (nb03 설계 때 실증).
5. **스모크.** `/smoke` 절차를 따른다. 구조화 출력 Pydantic 모델에는
   `extra="forbid"`, max_tokens는 여유 있게.
6. **가이드 노트북.** `.claude/rules/notebooks.md` 스타일로 한 편.
